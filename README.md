# claude-lk

Auto-sync context addon for Claude Code - no git dependency.

## What it does

- Generates `.lk` project context using AI (Gemini or Anthropic)
- Syncs automatically on Claude Code session end
- Provides instant project understanding to Claude
- Smart ignore patterns (global + project-specific)
- Zero configuration after setup
- Encrypted context protected by license

## Install

```bash
curl -fsSL https://github.com/jordi-zaragoza/claude-lk-releases/releases/download/v1.3.4/install.sh | bash
```

## Setup

```bash
claude-lk activate  # Enter license key
claude-lk setup     # Configure AI provider + Claude Code
claude-lk sync      # Initial sync (generates context + ignore)
```

## Usage

### With Claude Code

After setup, just use Claude Code normally:

```bash
claude
```

Context syncs automatically when you finish a session.

### Commands

```bash
claude-lk activate       # Activate license
claude-lk setup          # Configure AI + Claude Code
claude-lk sync           # Manual sync
claude-lk status         # Show status
```

## How it works

```
.claude/lk/
├── syntax.lk      # LK grammar (encrypted)
├── project.lk     # Project metadata (encrypted)
├── ignore         # Project-specific ignore patterns
└── domains/
    ├── cli.lk     # CLI domain files (encrypted)
    └── core.lk    # Core domain files (encrypted)
```

### Ignore Patterns

**Global patterns** (in config):
- `**/node_modules/**`, `**/dist/**`, `**/.git/**`, `**/*.lock`

**Project patterns** (`.claude/lk/ignore`):
- Auto-generated by AI on first sync
- Only project-specific patterns not covered by global

## MCP Server

Claude Code configuration (auto-configured by setup):

```json
{
  "mcpServers": {
    "claude-lk": {
      "command": "claude-lk",
      "args": ["serve"]
    }
  }
}
```
